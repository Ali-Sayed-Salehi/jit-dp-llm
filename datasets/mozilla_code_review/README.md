# Mozilla Code Review Dataset

This directory contains Mozilla Mercurial commits, model risk predictions, and
Phabricator code-review data linked at the per-commit level. The extraction
pipeline lives in `data_extraction/conduit/`.

## Files

- `all_commits.jsonl`
  - One Mercurial commit per line.
  - Fields: `node`, `desc`, `date`, `parents`.
  - Must be parent-before-child ordered. The Conduit scripts use this as the
    Mercurial graph order.

- `risk_predictions_eval.json`
  - Model predictions for the eval split.
  - Prediction rows are under `samples`.
  - Each row includes `commit_id`, `true_label`, `prediction`, and
    `confidence`.
  - `confidence` is interpreted as confidence in the predicted class, so the
    risk score is `confidence` when `prediction == 1` and `1 - confidence`
    when `prediction == 0`.

- `risk_predictions_final_test.json`
  - Model predictions for the final-test split.
  - Prediction rows are under `results`.
  - Each row includes `commit_id`, `true_label`, `prediction`, and
    `confidence`.
  - `confidence` is interpreted as the probability of label `1`.

- `per_commit_risk_scores.jsonl`
  - Generated by `data_extraction/conduit/get_commit_risk_scores.py`.
  - Expands row-level prediction scores to individual commits in contiguous
    same-bug blocks.
  - Top-level fields:
    - `commit_id`: Mercurial commit hash.
    - `risk_score`: probability in `[0, 1]` that the commit has label `1`.
    - `desc`: Mercurial commit description.

- `per_commit_drevs.jsonl`
  - Generated by `data_extraction/conduit/get_per_commit_drevs.py`.
  - One row per eligible commit with a trailing Mozilla Phabricator
    Differential Revision URL whose DREV is published and closed.
  - Top-level fields:
    - `commit_id`: Mercurial commit hash.
    - `dataset_split`: either `eval` or `final test`.
    - `risk_score`: copied from `per_commit_risk_scores.jsonl`.
    - `drev`: raw Conduit DREV object returned by
      `differential.revision.search`.

- `per_commit_drev_transactions.jsonl`
  - Generated by `data_extraction/conduit/get_drevs_transactions.py`.
  - One row per `per_commit_drevs.jsonl` row.
  - Top-level fields:
    - `commit_id`: Mercurial commit hash.
    - `drev_id`: numeric Differential Revision id.
    - `dataset_split`: copied from `per_commit_drevs.jsonl`.
    - `risk_score`: copied from `per_commit_drevs.jsonl`.
    - `transactions`: raw Conduit transaction objects returned by
      `transaction.search`.

- `drev_review_data.jsonl`
  - Generated by `data_extraction/conduit/create_code_review_dataset.py`.
  - Final compact review dataset, one row per commit/DREV.
  - Top-level fields:
    - `commit_id`: Mercurial commit hash.
    - `dataset_split`: either `eval` or `final test`.
    - `risk_score`: copied from the upstream per-commit DREV row.
    - `drev_submission_date`: ISO-8601 UTC timestamp for the DREV creation
      transaction when available.
    - `drev_author`: DREV author PHID when available.
    - `files_changed`: changed file paths from `hg status --change`.
    - `reviews`: extracted non-empty Phabricator comments, sorted by timestamp,
      with `author`, `submission_date`, and `comment`.

## Score Expansion

`get_commit_risk_scores.py` loads both prediction files, normalizes their
confidence formats into a single `risk_score`, then expands each scored commit
backward across contiguous commits with the same leading `Bug <id>` marker.
This mirrors the net-diff grouping used by
`data_extraction/mercurial/link_bug_diffs.py`.

Only commits in the valid interval from the eval split's first Mercurial commit
through the final-test split's last Mercurial commit are considered. Commits in
that interval with neither a direct prediction score nor an inherited
same-bug-block score are skipped.

Revert and backout commits are excluded from same-bug expansion.

## Split Assignment

`get_per_commit_drevs.py` derives eval and final-test boundaries by sorting the
prediction commit ids in `all_commits.jsonl` order:

- eval boundary: first to last eval prediction commit in Mercurial order.
- final-test boundary: first to last final-test prediction commit in Mercurial
  order.

The script only scans commits from the eval start boundary through the
final-test end boundary. It keeps commits that:

- have a leading `Bug <id>` marker;
- have a risk score in `per_commit_risk_scores.jsonl`;
- have a trailing `https://phabricator.services.mozilla.com/D...` URL;
- resolve to a published, closed DREV.

For split labeling, direct membership in a prediction file takes precedence. For
non-sample commits inside a boundary, the boundary determines the split. If the
eval and final-test boundaries overlap, nearest prediction sample in Mercurial
order is used as a deterministic tie-breaker.

## Regenerating

From the repo root:

```bash
source venv/bin/activate

python data_extraction/conduit/get_commit_risk_scores.py

python data_extraction/conduit/get_per_commit_drevs.py \
  --rate-limit-min-interval 0.5 \
  --max-retries 5 \
  --retry-base-sleep 5.0

python data_extraction/conduit/get_drevs_transactions.py \
  --page-limit 100 \
  --rate-limit-min-interval 0.5 \
  --max-retries 5 \
  --retry-base-sleep 5.0

python data_extraction/conduit/create_code_review_dataset.py \
  --hg-pull-max-attempts 5 \
  --hg-pull-retry-base-sleep 5.0
```

For a bounded smoke test:

```bash
python data_extraction/conduit/get_per_commit_drevs.py --debug
python data_extraction/conduit/get_drevs_transactions.py --debug
python data_extraction/conduit/create_code_review_dataset.py --debug
```

The Conduit stages require `CONDUIT_API_TOKEN` in `secrets/.env`. The final
assembly stage requires `hg` and a local Mozilla autoland clone under
`data_extraction/mercurial/repos/autoland`; it will clone or update that repo by
default. Use `--skip-repo-update` when the local clone already contains the
needed commits and network access should be avoided.
